# wbhack_dec25
Решение для ML-хакатона по определению дубликатов товаров на маркетплейсах

# Задание

## Отборочный этап

В отборочном этапе участникам предстоит решить задачу матчинга товаров — определить, являются ли два товара дубликатами (идентичными, одинаковыми товарами). Такая задача крайне важна для маркетплейса: корректное объединение одинаковых позиций улучшает качество каталога, повышает точность поиска и делает пользовательский опыт лучше.

Для обучения и проверки решений используются реальные данные маркетплейса - названия, описания, характеристики и изображения товаров.

Вам предстоит продумать стратегии работы с текстами, признаками и изображениями, чтобы добиться наилучших результатов. Например, два товара могут отличаться словом в описании или названии, но при этом быть одинаковыми.  В отборочном этапе вам необходимо решить задачу для товаров из категории Красота, где названия и описания разных товаров могут быть очень похожи при том, что сами товары могут быть разными.

## Финальный этап

На втором финальном этапе вам предстоит расширить ранее построенные модели и адаптировать их к новым категориям. У вас будет 5 часов, чтобы дообучить ваши модели на двух новых категориях: “Аксессуары для электроники” и “Бытовая техника”. Реализуйте новые подходы на базе своих прошлых решений или предложите новую архитектуру, но учитывайте ограничение по времени.

## Данные

Участникам предоставляются данные в формате .parquet и .zip:

beauty_train.parquet (accessories_train.parquet и house_full_train.parquet) - тренировочные данные, для которых известно являются ли товары дубликатами или нет. Таблица с колонками: 
* id - уникальный идентификатор пары товаров;
* id1 - уникальный идентификатор первого товара из пары;
* parentname1 - категория первого товара из пары;
* subjectname1 - подкатегория первого товара из пары;
* title1 - название первого товара, как указал продавец в карточке товара;
* description1 - описание первого товара, как указал продавец в карточке товара;
* characteristics1 - список видимых словарей-характеристик из карточки первого товара. Каждый словарь состоит из трех элементов с ключами: Name, Values, Value: 
“charcName” - название характеристики,
“charcValues” - список значений, которые продавец выставил для данной категории, 
“value” - числовое значение характеристики.
* id2 - уникальный идентификатор второго товара из пары; 
* parentname2 - категория второго товара из пары
* subjectname2 - подкатегория второго товара из пары
* title2 - название второго товара, как указал продавец в карточке товара;
* description2 - описание второго товара, как указал продавец в карточке товара;
* characteristics2 - список видимых словарей-характеристик из карточки о товара. Каждый словарь состоит из трех элементов с ключами: Name, Values, Value: 
"charcName” - название характеристики,
“charcValues” - список значений, которые продавец выставил для данной категории,
“value” - числовое значение характеристики,
* target - целевая переменная. 1 если товары являются дубликатами, иначе 0

beauty_test.parquet (accessories_test.parquet и house_full_test.parquet) - тестовые данные, на которых будет проверяться ваша модель. Содержит все те же колонки, что и тренировочные данные за исключением целевой переменной.
